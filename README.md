# AI Social World Simulator

Current focus: **social simulation layer only**.

This branch runs a Red Cliff sandbox with:

- no player interaction layer
- no narrative interpreter layer
- all character actions generated by LLM from current state
- optional hierarchical orchestration (main-event chat + sub-event chats)

## Run

```bash
npm start
```

Startup prints:
- whether API key is active
- endpoint
- model
- temperature
- history dump file path
- LLM trace IO directory (if enabled)
- orchestration mode and sqlite db path

Setup:

```bash
cp .env.example .env.local
# edit .env.local with your real key
npm start
```

Debug mode (decision drivers + utterances):

```bash
DEBUG_MODE=1 npm start
```

By default in debug mode, the simulator saves complete prompt + raw LLM output for each request under a temp trace directory.
You can control it with:

```bash
SIM_TRACE_LLM_IO=1
SIM_TRACE_DIR=/tmp/world-simulator/llm-trace
```

For faster iteration, cap per-tick actor count:

```bash
SIM_STEPS=3 SIM_MAX_ACTORS_PER_TICK=4 SIM_LLM_CONCURRENCY=4 DEBUG_MODE=1 npm start
```

For stronger same-phase causality, keep sequential planning (default):

```bash
SIM_PLANNING_MODE=sequential
SIM_ACTIVE_RATIO_PER_TICK=0.45
```

Determinism + randomness can be tuned explicitly:

```bash
SIM_RANDOM_SEED=42
SIM_DECISION_NOISE=0.15
SIM_BATTLE_NOISE=0.2
```

- `SIM_RANDOM_SEED`: same seed reproduces the same stochastic rolls.
- `SIM_DECISION_NOISE`: randomness in persuasion/agreement outcomes.
- `SIM_BATTLE_NOISE`: randomness in battle outcome probability.
- `SIM_ORCHESTRATION_MODE=hierarchical`: main-event planner + sub-event summarizer loop.
- `SIM_DB_PATH`: SQLite file for world/agents/relations/events/chat logs.
- Main-chat memory policy (fixed): after 3 rounds, use `summary + latest 3 rounds raw`.

Example `.env.local` for Kimi (OpenAI-compatible endpoint):

```bash
USE_LLM=1
LLM_API_KEY=your_kimi_key
LLM_BASE_URL=https://api.moonshot.cn/v1
LLM_ENDPOINT=/chat/completions
LLM_MODEL=moonshot-v1-8k
LLM_TEMPERATURE=1
LLM_MAX_TOKENS=900
LLM_TIMEOUT_MS=30000
LLM_RETRY_COUNT=3
LLM_RETRY_BACKOFF_MS=600
SIM_STEPS=8
SIM_MAX_ACTORS_PER_TICK=0
SIM_LLM_CONCURRENCY=4
DEBUG_MODE=0
```

Note: if not specified, Moonshot/Kimi providers default to `temperature=1` in this project.
If you see `This operation was aborted`, reduce concurrency and increase timeout, for example:

```bash
SIM_STEPS=3 SIM_MAX_ACTORS_PER_TICK=4 SIM_LLM_CONCURRENCY=2 LLM_TIMEOUT_MS=45000 DEBUG_MODE=1 npm start
```

If you see JSON cut off in the middle, increase `LLM_MAX_TOKENS` (recommended `700-900`).
Each run also writes a temporary markdown history file under system temp dir (or `SIM_HISTORY_DIR` if set).
`history_brief` is rolling and updates from recent events; default window is last 2 days (`SIM_HISTORY_BRIEF_DAYS=2`).
You can also tune expression richness:

```bash
SIM_SPEECH_MAX_CHARS=80
SIM_SUMMARY_MAX_CHARS=120
SIM_RATIONALE_MAX_CHARS=160
```

## Structure

- `src/socialMain.js`: start entry for social sandbox
- `src/social/config.js`: runtime config and key masking
- `src/social/seedRedCliff.js`: major Red Cliff characters and initial state
- `src/social/actionPlanner.js`: LLM-generated action planning per character
- `src/social/stateUpdater.js`: state transition/update engine
- `src/social/engine.js`: multi-step social simulation loop
- `src/social/reporter.js`: output report and debug traces
- `test/simulator.test.js`: existing base tests
